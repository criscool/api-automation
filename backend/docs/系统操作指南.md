API 自动化测试平台是一个 **AI 多智能体驱动** 的接口自动化测试工具，核心能力：

- **上传接口文档**：支持 OpenAPI / Swagger / Postman / PDF 格式
- **AI 智能分析**：7 个 AI 智能体协作，自动分析接口依赖、生成测试用例和 pytest 脚本
- **一键执行测试**：生成 Allure 可视化报告，支持定时任务

---

## 二、如何导入接口文件生成用例

### 方式一：文档工作流（AI 智能分析）

1. 进入「接口自动化」→「文档工作流」
2. 上传接口文档（支持 OpenAPI / Swagger / Postman / PDF 格式）
3. 系统自动解析文档，经过 **7 个 AI 智能体协作**生成 pytest 测试脚本
4. 生成的脚本保存在「用例管理」中，可直接执行

### 方式二：上传依赖 JSON 直接生成

1. 进入「接口自动化」→「文档工作流」→「依赖导入」
2. 上传已分析好的依赖关系 JSON 文件
3. 系统直接根据依赖关系生成测试用例和 pytest 脚本
4. 适用于已完成接口分析、只需生成用例的场景

---

## 三、执行测试

1. 进入「用例管理」，勾选要执行的用例
2. 点击「批量执行」或单独执行某条用例
3. 在「执行报告」页面查看实时进度和结果
4. 点击「预览」查看 Allure 可视化报告

---

## 四、环境配置

测试环境通过 `generated_tests/automation/core/config/env/` 目录下的 YAML 文件配置：

- `test.yaml` — 测试环境
- `staging.yaml` — 预发布环境
- `production.yaml` — 生产环境

执行测试时通过 `--env` 参数指定环境。

---

## 五、定时任务

进入「接口自动化」→「定时任务」，可创建定时执行计划：

1. 选择要执行的用例分类或脚本
2. 设置执行时间（每天 / 每周 / 仅一次）
3. 系统按计划自动执行测试并生成报告
